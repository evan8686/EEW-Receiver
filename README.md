EEW-Receiver (地震预警接收器)
📌 项目初衷
由于目前大陆主流地震预警系统尚未全面接入 台湾地区中央气象署（CWA） 的预警数据，福建及浙江南部沿海城市在面对台湾地震带来的明显震感时，往往无法及时获得预警推送。为了填补这一空白，确保沿海居民能第一时间采取避险措施，特别开发了本项目。

🛠️ 项目性质
开发背景：本项目纯属个人兴趣驱动，在 Gemini AI 的辅助下开发完成。

技术架构：使用 Kotlin 编写，通过 WebSocket 实时订阅 Wolfx API 推送。

免责声明：本人暂无软件专业开发经验，本项目目前以自用和技术测试为目的。本人不对 API 接口的稳定性、准确性及预警延迟负责。

📖 使用说明
设置阈值：打开 App 进入设置页面，可自定义触发全屏警报的震级阈值（默认为 3.0 级）。

权限授予：请务必授予应用所有必要的通知权限，并配置系统“防杀后台”相关设置。

运行状态：

监控中：未收到预警时，通知栏会常驻“地震预警监控中”通知以维持后台活跃。

普通通知：收到预警但未达到设定的震级阈值时，仅发送静默文本通知。

全屏警报：收到预警且达到阈值时，将触发全屏弹窗、高分贝警报音及震动。

🧪 功能测试流程
为确保预警系统在关键时刻有效，请按照以下步骤测试：

1. 桌面状态测试
点击设置页面的 “模拟触发” 按钮，立即返回手机桌面。

等待约 3 秒，确认是否出现全屏警报界面、声音及震动，并检查通知栏是否有测试通知。

2. 锁屏/息屏状态测试
点击 “模拟触发” 后，立即返回桌面并在 3 秒内按下电源键息屏。

等待并确认系统能否成功穿透锁屏界面弹出全屏警告。

⚠️ 关键配置（以 OPPO ColorOS 为例）
若测试发现没有通知栏预警通知，也没有全屏弹窗，请严格按下列方式操作：

1. 设置 - 通知 - EEW Receiver

a. 开启“允许通知”，并将 “锁屏/横幅” 打勾。

b. 打开 “铃声/震动/允许打扰”（该项指即便手机处于免打扰模式时，应用仍能正常响铃和震动）。

c. 将“类别”下的 “地震预警事件” 设为允许通知，“地震预警后台监控”则维持默认的“静默通知”即可。

2. 设置 - 应用管理 - EEW Receiver - 权限管理

a. 锁屏显示：设为 “允许”。

b. 点击右上角 3 个点，选择 “解锁所有授权限制” 进行解除。

c. 滚动页面到下方，进入 “其他权限” -> “特殊应用权限”：

悬浮窗：设为“允许”。

后台弹出界面：设为“允许”。

发送全屏通知：设为“允许”。


🚧 优化方向与注意事项
欢迎各位大神针对以下方向进行优化：

[ ] 距离判断：目前收到推送直接触发，不判断所在地与震中的距离。

[ ] 烈度测算：目前无法进行震波到达倒计时及本地烈度测算。

[ ] 地图功能：目前没有内置显示震中位置的地图功能。

[ ] 数据源说明：当前内建默认源为 台湾中央气象署（用户可根据需求自行修改订阅源）。
